# M3 Linear Seeding Pack

最后更新：2026-06-28

这个文件用于把 `M3 · 规则沉淀与复盘` 从 PRD 拆成可执行 issue，并作为同步到 Linear 的依据。

## M3 目标

让系统从真实使用中持续增强，把一次次任务中的有效经验沉淀成可复用规则，而不是停留在聊天记录里。

## M3 对应 PRD 交付

- 任务后复盘
- 规则候选生成
- 用户确认保存
- 规则分级与版本
- 客户端同步提示

## 建议创建的 Issues

1. `M3-01 · 任务后复盘入口与复盘表单`
2. `M3-02 · 规则候选生成与分类`
3. `M3-03 · 候选规则确认保存到 AI-OS`
4. `M3-04 · 规则轻量版本与变更摘要`
5. `M3-05 · 客户端同步提示与收尾验收`

## 建议执行顺序

1. 先把复盘入口做进 `AI-OS` 页面，建立稳定触发点
2. 再根据复盘内容生成规则候选，并做全局规则 / 工作流 / 决策记录分类
3. 再提供用户确认保存能力，避免系统直接写入共享规则
4. 在保存链路稳定后补轻量版本与变更摘要
5. 最后补客户端同步提示，确保新规则能回流到不同客户端使用面

## 为什么 M3 要这样拆

- 规则沉淀是复利核心，但如果一开始把“复盘、生成、确认、写入、同步”一次做完，范围会很快失控
- 先有入口，再有候选，再有保存，能确保每一步都有独立验收点
- 这种拆法也符合“候选先生成，再逐条确认保存”的产品原则
