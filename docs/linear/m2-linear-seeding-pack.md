# M2 Linear Seeding Pack

最后更新：2026-06-28

这个文件用于把 `M2 · 环境诊断与安全合并` 从 PRD 拆成可执行 issue，并作为下一阶段同步到 Linear 的依据。

## M2 目标

避免盲目覆盖用户已有配置，让用户在接入 AI 协作底座前，先知道自己的本地环境里已经有什么、可能冲突什么、哪些操作需要更谨慎。

## M2 对应 PRD 交付

- 本地环境检测
- 已有规则文件识别
- diff 预览
- 合并建议
- 敏感信息检查
- MCP 连接状态检测
- MCP 权限与失败兜底检查

## 建议创建的 Issues

1. `M2-01 本地环境检测与工作区健康检查`
2. `M2-02 已有规则文件识别与冲突扫描`
3. `M2-03 配置 diff 预览与合并建议`
4. `M2-04 敏感信息检查与风险提示`
5. `M2-05 MCP 连接状态检测与失败兜底`

## 建议执行顺序

1. 先做本地环境检测，建立“当前机器 / 当前项目 / 当前客户端配置”基线
2. 再识别已有规则文件，避免后续输出直接覆盖用户现有配置
3. 在此基础上补 diff 与合并建议
4. 再补敏感信息检查，降低误写入风险
5. 最后做 MCP 连接状态检测与失败兜底，让外部接入层可解释

## 为什么 M2 要这样拆

- 如果把“环境诊断与安全合并”一次做完，前后端、规则层、文件系统和 MCP 状态会耦在一起，验收会变得模糊
- 先建立本地环境基线，后续所有“识别 / diff / 合并 / 风险提示”才有可依赖的输入
- 这个拆法也更适合保留串行执行和证据闭环
