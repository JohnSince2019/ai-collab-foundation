# M4 Linear Seeding Pack

最后更新：2026-06-28

这个文件用于把 `M4 · ContentOps 联动` 从 PRD 拆成可执行 issue，并作为同步到 Linear 的依据。

## M4 目标

把 AI 协作底座作为 ContentOps 的上游模块，让 ContentOps 在真正生成内容前先吃到用户定位、边界、风格和素材上下文。

## M4 对应 PRD 交付

- 用户定位读取
- 内容边界读取
- 表达风格读取
- 高频素材来源读取
- 母稿生成上下文注入

## 建议创建的 Issues

1. `M4-01 · 用户定位读取与 Profile 映射`
2. `M4-02 · 内容边界读取与规则注入`
3. `M4-03 · 表达风格读取与风格卡生成`
4. `M4-04 · 高频素材来源读取与结构化映射`
5. `M4-05 · 母稿生成上下文注入与联动验收`

## 建议执行顺序

1. 先打通用户定位读取，形成最小 Profile 输入
2. 再接内容边界和表达风格，让生成前约束完整起来
3. 再接素材来源，补齐常用信息源
4. 最后注入母稿生成链路，并完成联动验收

## 为什么 M4 要这样拆

- ContentOps 联动的核心不是“大接入”，而是先把生成前最关键的上下文入口对齐
- 这五项正好对应 PRD 里的五个明确交付，拆开后更适合串行推进和验收
